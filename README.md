i have made a trekking management role-based app 
the roles introduced are  - 
1.Admin - main controller who controls the members(Staff and Users) on the app 
2.Staff - basically the guides and other staff 
3.User - The trackker which wants to get the service and came for trekking

Features - 
ADMIN :
  admin can add new staff members and also users registering them on the portal 
  admin can add new routes and assign staff 
  admin can track total treks,registered trekkers,total bookings
  also verify the new user or new staff member 
  admin can blaclist /ban or suspend the user and staff

STAFF :
  can update number of slots left 
  also status of trek(start completed open or closed )

USER:
  can see available treks 
  can see the status of registered trek and also cancel booking
